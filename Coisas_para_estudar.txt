O kernel é um componente fundamental de um sistema operacional que tem controle completo sobre o hardware e software do sistema. Ele é responsável por facilitar a interação entre os componentes de hardware e software, bem como controlar os recursos do sistema, como memória, processador, dispositivos de entrada/saída e sistemas de arquivos.

Algumas características e funcionalidades importantes do kernel são:

O kernel é o primeiro programa carregado na memória durante a inicialização do sistema operacional.
Ele gerencia os processos do sistema, alocando recursos de hardware para eles e garantindo que eles sejam executados de forma justa e eficiente.
O kernel também controla a memória do sistema, alocando e desalocando espaço para os processos e garantindo a proteção da memória entre os processos.
Além disso, o kernel é responsável pela comunicação entre os processos, permitindo a troca de dados e sincronização entre eles.
O kernel também fornece suporte para dispositivos de hardware, como drivers de dispositivo, permitindo a interação com periféricos.

The Unified Extensible Firmware Interface (UEFI, successor of the EFI) is an interface between operating systems and firmware. It provides a standard environment for booting an operating system and running pre-boot applications.

O firmware é um tipo de software embutido em um hardware específico, como um dispositivo eletrônico ou um componente de um sistema. Ele é responsável por controlar e fornecer instruções permanentes para o hardware, permitindo que ele se comunique com outros dispositivos e execute funções básicas de entrada/saída.

O firmware é armazenado em uma memória não-volátil, como ROM (Read-Only Memory), EPROM (Erasable Programmable Read-Only Memory) ou flash memory. Esses tipos de memória garantem que as instruções do firmware sejam mantidas mesmo em caso de falha de energia.
O firmware difere dos drivers de dispositivo, pois está incorporado diretamente no hardware, enquanto os drivers são instalados no sistema operacional.
O firmware é responsável por inicializar o hardware e executar as funções básicas necessárias para o seu funcionamento.
Atualizações de firmware podem ser lançadas pelos fabricantes para corrigir bugs, adicionar recursos ou melhorar o desempenho e a segurança do hardware.
Alguns exemplos de dispositivos que possuem firmware são: placas-mãe de computador, impressoras, roteadores, câmeras digitais, smartphones, TVs, entre outros.

O GRUB (GRand Unified Bootloader) é um gerenciador de boot amplamente utilizado em sistemas operacionais baseados em Unix, como o Linux. Ele é responsável por carregar o kernel do sistema operacional na memória e iniciar o processo de inicialização do sistema.
O GRUB é capaz de carregar vários sistemas operacionais instalados em um computador, permitindo ao usuário escolher qual sistema operacional inicializar durante o processo de inicialização.
Ele suporta diferentes sistemas de arquivos, como ext4, NTFS, FAT, entre outros, permitindo que o usuário acesse e inicialize a partir de partições formatadas em diferentes sistemas de arquivos.
O GRUB também suporta a configuração de várias opções de inicialização, como passar parâmetros para o kernel, definir o nível de execução do sistema (modo de usuário único, modo de recuperação, etc.) e configurar opções de inicialização avançadas.
Além disso, o GRUB possui uma interface de linha de comando que permite ao usuário executar comandos e realizar várias tarefas, como editar opções de inicialização, carregar módulos adicionais e solucionar problemas de inicialização.

Existem IPs públicos e privados. Os endereços privados são usados internamente (tipo em casa) - um roteador é a interface que separa o interno (2 pc's, 4 celulares, 3 smartwatches por exemplo) do mundo externo. Então o roteador da casa atribui um endereço de IP privado a cada aparelho que se conecta a ele. E o roteador em si tem um endereço de IP público atribuído a ele. Ou seja, a casa é vista como 1 endereço público, sendo que dentro da casa existem 9 IPs privados. O endereço IP privado do seu dispositivo será traduzido em NAT (Network Address Translation) para o endereço público do roteador.

Um gateway é um nó de rede usado em telecomunicações que conecta duas redes com protocolos de transmissão diferentes. Ele funciona como um ponto de entrada e saída para uma rede, pois todos os dados devem passar ou se comunicar com o gateway antes de serem roteados. Em redes baseadas em IP, o único tráfego que não passa por pelo menos um gateway é o tráfego entre nós no mesmo segmento de rede local (LAN). O termo "default gateway" ou "network gateway" também pode ser usado para descrever o mesmo conceito[0].

Os gateways têm algumas características e funcionalidades importantes:

Simplificam a conectividade com a internet, permitindo que vários dispositivos em uma rede se comuniquem com a internet por meio de um único dispositivo.
Podem atuar como servidores proxy e firewalls, fornecendo recursos adicionais de segurança e controle de acesso.
São capazes de converter protocolos, facilitando a compatibilidade entre diferentes protocolos de rede.
Podem operar em qualquer camada do modelo de referência OSI (Open Systems Interconnection), dependendo das necessidades específicas da rede[4][5].
O gateway é implementado na borda de uma rede e gerencia todos os dados direcionados interna ou externamente a essa rede. Quando uma rede deseja se comunicar com outra, o pacote de dados é enviado para o gateway e, em seguida, roteado para o destino através do caminho mais eficiente. Além de rotear dados, um gateway também armazena informações sobre os caminhos internos da rede hospedeira e os caminhos de quaisquer redes adicionais encontradas[5].

Um gateway padrão é um dispositivo de rede que atua como o ponto de entrada ou saída para uma rede. Ele é responsável por encaminhar pacotes de dados entre diferentes redes, como a rede local (LAN) e a internet. Quando um dispositivo em uma rede precisa se comunicar com outro dispositivo em uma rede diferente, os pacotes de dados são enviados para o gateway padrão, que os roteia para o destino correto.

O gateway padrão também pode ser chamado de roteador padrão ou rota padrão. Ele é configurado nos sistemas operacionais dos dispositivos conectados à rede e é usado para direcionar o tráfego de rede para fora da rede local. O endereço do gateway padrão é geralmente o endereço IP do roteador que está conectado à rede.

O DNS (Domain Name System) é um sistema que traduz nomes de domínio em endereços IP. Ele funciona como um diretório telefônico da internet, associando os nomes de domínio (como "google.com") aos seus respectivos endereços IP (como "172.217.28.174"). Isso permite que os navegadores e outros aplicativos acessem recursos da internet usando nomes de domínio fáceis de lembrar, em vez de ter que digitar endereços IP numéricos.

O DNS é essencial para a navegação na internet, pois simplifica o acesso a recursos online. Sem o DNS, seria necessário memorizar os endereços IP de todos os sites que desejamos acessar. Com o DNS, podemos digitar um nome de domínio em um navegador e o sistema se encarrega de traduzir esse nome em um endereço IP.

O sistema DNS é composto por diversos tipos de servidores, como servidores raiz, servidores de autoridade e servidores recursivos. Cada um desses servidores desempenha um papel importante na resolução de nomes de domínio em endereços IP. Quando um usuário digita um nome de domínio em seu navegador, o computador envia uma solicitação para o servidor DNS mais próximo, que por sua vez consulta outros servidores DNS até encontrar o endereço IP correspondente ao nome de domínio.

Além de traduzir nomes de domínio em endereços IP, o DNS também suporta recursos avançados, como redirecionamentos, registros MX para e-mails, registros de serviço (SRV) para aplicativos específicos e muito mais. Ele é um dos protocolos mais usados na internet e é essencial para a comunicação e conectividade online[7][11].

Existem diversos provedores de serviços DNS, cada um com suas características e particularidades. Alguns dos DNS mais conhecidos são:

Google DNS: 8.8.8.8 e 8.8.4.4
Cloudflare DNS: 1.1.1.1 e 1.0.0.1
OpenDNS: 208.67.222.222 e 208.67.220.220
Esses provedores de serviços DNS podem oferecer recursos adicionais, como segurança, velocidade de resposta e filtragem de conteúdo. A escolha do DNS a ser utilizado depende das necessidades e preferências do usuário[4].

Em resumo, o DNS é um sistema que traduz nomes de domínio em endereços IP, permitindo que os usuários acessem recursos na internet de forma fácil e intuitiva. Ele é uma parte essencial da infraestrutura da internet e facilita a navegação e a conectividade online[11].
